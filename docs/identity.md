# contracts 身份

## 我是谁

`contracts` 是 FoundationX 的**跨域稳定契约层**。它定义跨域端口、事件协议和 DTO 契约，是各域之间唯一的编译期接口依赖。

## 我做什么

| 职责 | 说明 |
|------|------|
| 跨域端口 | 定义各域之间的稳定 Go interface（窄接口，3-5 方法） |
| 事件协议 | 定义跨域事件类型、Topic 名称和 payload schema |
| DTO 契约 | 定义跨域传输对象的不可变结构 |
| 版本契约 | 定义接口版本号和兼容性规则 |

## 我不做什么

| 不是 | 原因 |
|------|------|
| **不是域内接口** | 域内接口留在各域内部模块 |
| **不是领域值对象** | 领域值对象属于 L2.5（decimalx / domain-*） |
| **不是业务实现** | contracts 只定义接口，不实现业务逻辑 |
| **不是模板源** | 模板生成属于 xlib-standard |
| **不依赖 L1 运行时** | contracts 在依赖层次上高于 L1 |

## 我的边界

```
我拥有:
  - 跨域 Go interface 定义
  - 事件 Topic 名称和 payload 类型
  - 跨域 DTO 结构体
  - 接口版本号管理

我不拥有:
  - 域内接口（属于各域内部）
  - 领域值对象（属于 L2.5）
  - 接口实现（属于各域模块）
  - L1 运行时依赖
```

## 宪法合规

| 条款 | 遵循方式 |
|------|----------|
| §1 P7 | contracts 只定义跨域稳定契约 |
| §3.2 | 单向下行依赖 — 上层依赖 contracts，contracts 不依赖上层 |
| §3.4 | 禁止依赖 L1 运行时和业务域实现 |
| §4.1 | 窄接口（3-5 方法），编译期检查 |
